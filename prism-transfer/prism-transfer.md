# Skill 3: Prism Transfer — Перенос призм с конкурента на товар-герой

> **Полный рабочий промпт.** Скопируй целиком и вставь как системный промпт или первое сообщение в LLM.  
> Затем приложи нужные файлы (см. раздел «Входные данные») и напиши: **«Запусти Prism Transfer»**.

---

## 📋 ИНСТРУКЦИЯ ДЛЯ ЗАПУСКА (прочитай сначала)

### Что это и зачем

Этот инструмент — третий шаг в цепочке анализа. Он берёт готовые призмы конкурента (результат Skill 2) и сравнивает их с данными твоего товара-героя. Цель: найти, какие аудиторные гипотезы конкурента применимы к твоему товару, а какие нет. Это не просто «копирование» кластеров — это фильтрация через реальные отзывы товара-героя. Результат — **экспериментальные призмы**: новые гипотезы о покупателях, которые ты бы не обнаружила без данных конкурента.

### Что нужно подготовить (входные данные)

**Обязательно:**

1. **`[конкурент]-prisms-library.md`** — файл накопителя призм конкурента (Секция 7 из результата Skill 2 по конкуренту). Содержит: кластеры, драйверы, стоп-факторы, серую зону, Conflict Map, калибровку.

2. **`signal-markup.md` товара-героя** — результат Skill 1 по твоему товару (таблица отзывов с размеченными сигналами). Это основной источник доказательной базы.

**Желательно:**

3. **`perception-prisms.md` товара-героя** — полный результат Skill 2 по твоему товару. Нужен для контекста: какие кластеры у тебя уже есть, чтобы не дублировать их как «новые».

### Что ты получишь на выходе

- **Секция A:** Таблица переноса — краткий итог по каждому механизму
- **Секция B:** Карточки экспериментальных призм — детальные гипотезы с доказательствами из твоих отзывов
- **Секция C:** Отклонённые переносы — что проверялось, но не подтвердилось (и почему)
- **Секция D:** Готовый блок для добавления в `prisms-library.md` товара-героя

### Пошагово: что вставить в LLM и в каком порядке

1. Скопируй **весь этот файл** и вставь его как первое сообщение (системный промпт)
2. Приложи файл **`[конкурент]-prisms-library.md`**
3. Приложи файл **`signal-markup.md`** товара-героя
4. Приложи файл **`perception-prisms.md`** товара-героя (если есть)
5. Напиши: `Запусти Prism Transfer`
6. Если модель остановилась — напиши «продолжай»

Если анализируешь несколько конкурентов — прогоняй каждого **отдельно**, потом объединяй результаты вручную.

### Как читать и использовать результат

- **Начни с Секции A** (таблица) — быстрый обзор: что перенеслось, с какой уверенностью
- **Перейди к Секции B** (карточки) — детали по каждой экспериментальной призме: доказательства, ограничения, смысл для позиционирования
- **Просмотри Секцию C** — список того, что не перенеслось. Это тоже ценная информация: значит, у тебя нет этой аудитории (или данных недостаточно)
- **Секцию D** — скопируй и добавь в конец файла `prisms-library.md` товара-героя перед финальным прогоном Skill 2

> ⚠️ **Важно:** Экспериментальные призмы — это **гипотезы**, а не подтверждённые кластеры. Они требуют проверки в финальном прогоне Skill 2.

## КРИТИЧЕСКИ ВАЖНЫЕ ПРАВИЛА

### Правило 1: Калибровочные отзывы имеют наивысший приоритет

**Система тонов (унаследована из Skill 2):**

| Тон | Название | Маркеры |
|-----|----------|---------|
| А | Восторг | эмодзи 😍🥰🔥✨, «!!!», слова «бомба/обалден/шикар/кайф/находка» |
| Б | Обида | «увы/разочар/ужас/не возьму», негатив преобладает над позитивом |
| В | Трезвый | спокойный описательный тон, нет эмодзи/восклицаний, есть «но»/«однако»/«зато», фразы «в целом», «для меня», «неплохо» |

**Калибровочный отзыв** = тон В (трезвый) + содержит одновременно и плюсы, и минусы.

Почему они важнее:
- Вероятнее всего, это живой, незакупленный отзыв
- Автор рационален — видит обе стороны
- Описывает реальные компромиссы, а не эмоциональные полюса

**Для доказательства переноса** — и соответствующие уровни уверенности:

| Найдено | Уверенность | Символ |
|---------|------------|--------|
| ≥2 калибровочных отзыва (тон В, плюс+минус) | Сильная | ● |
| ≥3 любых информативных, но <2 калибровочных | Умеренная | ◐ |
| 1–2 отзыва любого тона | Слабый сигнал | ○ |

### Правило 2: Экспериментальные призмы ≠ подтверждённые кластеры

Никогда не называть экспериментальные призмы «аудиторными сегментами» или «кластерами». Это **гипотезы**, производные от данных конкурента. Статус — гипотеза до финального прогона Skill 2.

### Правило 3: Всегда цитировать ID отзывов

Каждое утверждение о товаре-герое должно быть подкреплено конкретными ID из `signal-markup.md`. Без ID — это домысел, а не вывод.

### Правило 4: Указывать источник каждой экспериментальной призмы

Для каждой призмы: из какого конкурента, из какого кластера/стоп-фактора/серой зоны конкурента она взята, через какой механизм получена.

### Правило 5: Проценты — запрещены

Только количество отзывов. Фраза «28% покупателей» — ошибка. Правильно: «7 отзывов из 64 информативных».

### Правило 6: Не переносить, если нет отзывов товара-героя

Если в данных товара-героя нет ни одного отзыва, подтверждающего гипотезу — пометить «не переносится» и занести в Секцию C.

### Правило 7: Не дублировать уже существующие кластеры

Если гипотеза совпадает с уже существующим кластером из `perception-prisms.md` товара-героя — не создавать новую призму. Вместо этого пометить: «совпадает с кластером [название], подтверждает его».

## МЕХАНИЗМ 1: ИНВЕРСИЯ СТОП-ФАКТОРОВ (Вариант Б — «агрессивный»)

**Приоритет: высший.**

### Логика

У конкурента есть стоп-факторы — то, что отталкивает его покупателей. Если наш товар-герой **не имеет этого минуса** или **имеет противоположное** — это потенциальный угол перетягивания аудитории.

### Шаги

Для каждого стоп-фактора из `КЛАСТЕРЫ → стоп_факторы_гипотеза` конкурента:

**Шаг 1.1: Формулировка инверсии**
> «У конкурента стоп-фактор: 🔴 [сигнал]. Есть ли у нас этот же минус? Или есть противоположное?»

**Шаг 1.2: Поиск контр-свидетельств в отзывах товара-героя**

Искать в `signal-markup.md` отзывы, где:
- Нет этого минуса (но человек описывает тот же сценарий использования)
- Есть явный позитив по этому параметру

**Приоритет поиска:** сначала калибровочные отзывы (тон В с плюсами И минусами), затем остальные.

**Шаг 1.3: Оценка доказательной базы**

| Найдено | Уверенность | Статус |
|---------|------------|--------|
| ≥2 калибровочных отзыва (тон В, плюс+минус) | ● Сильная | ✅ Создать экспериментальную призму «переманивание» |
| ≥3 любых информативных, но <2 калибровочных | ◐ Умеренная | ✅ Создать экспериментальную призму «переманивание» (слабое основание) |
| 1–2 отзыва любого тона | ○ Слабый сигнал | ⚠️ Недостаточно данных — не переносить |
| 0 отзывов | — | ❌ Не переносится — занести в Секцию C |

**Шаг 1.4: Создать карточку призмы**

Если статус ✅ — создать экспериментальную призму типа `poaching-angle` (переманивание). Заполнить шаблон из Секции B.

## МЕХАНИЗМ 2: ЗЕРКАЛО (Вариант А — «надёжный»)

**Приоритет: средний.**

### Логика

У конкурента есть кластеры — группы покупателей с определёнными пучками сигналов. Если у нашего товара-героя есть отзывы с теми же пучками — это означает, что та же аудитория есть и у нас, но мы, возможно, не знали об этом.

### Шаги

Для каждого кластера из `КЛАСТЕРЫ` конкурента:

**Шаг 2.1: Извлечение драйверов**

Список `драйверы` кластера конкурента.

**Шаг 2.2: Поиск совпадающих сигналов в отзывах товара-героя**

Искать в `signal-markup.md` отзывы, где присутствуют 2+ совпадающих 🟢 сигнала из пучка конкурента.

**Шаг 2.3: Оценка доказательной базы**

> Механизм 2 требует более высокого порога доказательства, чем Механизм 1, поскольку здесь мы переносим целый кластер (пучок сигналов), а не просто инвертируем один стоп-фактор. Зеркало — «надёжный» вариант.

| Найдено | Уверенность | Статус |
|---------|------------|--------|
| ≥3 калибровочных отзыва (тон В, плюс+минус) | ● Сильная | ✅ Подтверждённая экспериментальная призма |
| ≥3 любых информативных, но <3 калибровочных | ◐ Умеренная | ✅ Экспериментальная призма (слабое основание) |
| 1–2 отзыва | ○ Слабый сигнал | ⚠️ Недостаточно данных — не переносить |
| 0 отзывов | — | ❌ Нет совпадения — занести в Секцию C |

**Шаг 2.4: Проверка на дубликат**

Если в `perception-prisms.md` товара-героя уже есть кластер с теми же драйверами — не создавать новую призму. Записать: «совпадает с кластером [название] — подтверждает его, уверенность повышается».

**Шаг 2.5: Создать карточку призмы (если не дубликат)**

Если статус ✅ и не дубликат — создать призму типа `mirror` (зеркало). Заполнить шаблон из Секции B.

## МЕХАНИЗМ 3: ПЕРЕНОС СЕРОЙ ЗОНЫ (Вариант В — «тонкий»)

**Приоритет: низший.**

### Логика

«Серая зона» — это молчаливые покупатели, которые купили продукт, несмотря на его недостатки. В отзывах конкурента это проявляется как отзывы с одновременным наличием плюсов И минусов (компромиссные отзывы). Если наш товар-герой устраняет именно тот компромисс, который принял покупатель конкурента — это сильный угол.

### Шаги

Для каждого кластера конкурента с описанием серой зоны (`СЕРАЯ ЗОНА` в карточках кластера), или для отзывов конкурента с тоном В (трезвый), содержащих одновременно плюс и минус:

**Шаг 3.1: Формулировка компромисса**
> «Покупатель конкурента принял компромисс: 🟢 [плюс], несмотря на 🔴 [минус]. Может ли наш товар убрать этот минус?»

**Шаг 3.2: Только компромиссные отзывы**

Работать **исключительно** с отзывами, содержащими одновременно плюс И минус. Чисто позитивные и чисто негативные отзывы конкурента — игнорировать в этом механизме.

**Шаг 3.3: Поиск в отзывах товара-героя**

Искать в `signal-markup.md` отзывы, где:
- Присутствует 🟢 тот же плюс (что и у конкурента)
- Отсутствует 🔴 тот же минус (или есть явный позитив по этому параметру)

**Шаг 3.4: Оценка доказательной базы**

| Найдено | Уверенность | Статус |
|---------|------------|--------|
| ≥2 отзыва с 🟢 плюсом и отсутствием 🔴 минуса | ◐ Умеренная | ✅ Создать призму «устранение компромисса» |
| 1 отзыв | ○ Слабый сигнал | ⚠️ Недостаточно данных — не переносить |
| 0 отзывов | — | ❌ Не переносится — занести в Секцию C |

**Шаг 3.5: Создать карточку призмы**

Если статус ✅ — создать призму типа `compromise-elimination` (устранение компромисса). Заполнить шаблон из Секции B.

## ПРИМЕЧАНИЕ О КОНКУРЕНТАХ

### Прямые конкуренты

Инверсия стоп-факторов (Механизм 1) особенно ценна для прямых конкурентов: если их покупатели уходят из-за конкретного минуса, а у нас этого минуса нет — это прямой угол для перетягивания.

### Косвенные конкуренты

Зеркало (Механизм 2) и перенос серой зоны (Механизм 3) особенно ценны для косвенных конкурентов: они расширяют картину аудитории за пределы нашей категории. Но будь осторожна: одинаковый сигнал в разных категориях может означать разные вещи.

### Обработка нескольких конкурентов

Прогоняй каждого конкурента **отдельным запуском**. В итоговый файл `prisms-library.md` добавляй блоки последовательно — по одному блоку на конкурента. Если одна и та же экспериментальная призма возникла при анализе двух конкурентов — это повышает её уверенность.

## ЧАСТЫЕ ВОПРОСЫ

**Сколько экспериментальных призм обычно получается?**  
2–5. Больше — вероятно, занижены стандарты доказательства. Меньше одной — либо конкурент очень непохожий, либо у товара-героя мало отзывов.

**Что делать, если у конкурента нет стоп-факторов в prisms-library.md?**  
Посмотри в полном выводе Skill 2 конкурента (Секция 2, карточки кластеров — поле `стоп_факторы_гипотеза`). Если данных нет вообще — используй только Механизм 2 и 3.

**Нужно ли прогонять Prism Transfer после каждого нового конкурента?**  
Да, отдельно для каждого конкурента. Результаты накапливаются в `prisms-library.md`.

**Что если экспериментальная призма совпадает с уже существующим кластером?**  
Не создавать дубликат. Записать: «совпадает с кластером [название] товара-героя — подтверждает его». Это повышает уверенность существующего кластера, но не создаёт новой призмы.

**Когда экспериментальная призма станет «настоящим» кластером?**  
После финального прогона Skill 2 по товару-герою с флагом `prisms-library.md`. Если подтверждается ≥3 отзывами — становится кластером ◐. Если ≥5 — кластером ●.
